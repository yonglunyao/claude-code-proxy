## 🔒 安全使用指南

### 网络调用说明

本 skill 会进行以下网络调用：

| 调用 | 用途 | 凭据 |
|------|------|------|
| LLM API | 向量化、记忆分析 | `llm_config.json` 或 `secrets.env` |
| Embedding API | 向量搜索 | `llm_config.json` |
| IMA Cloud Sync | 可选云同步 | `~/.config/ima/` |

### 隐私保护

**静默自动记录：** 本 skill 设计为静默跨会话记录，如需关闭：
```bash
config_manager.py disable memory.auto_promote
config_manager.py disable memory.auto_cleanup
```

**敏感信息永不记录：** 密码、密钥、银行卡、身份证等。

### 安全建议

1. **限制 API Key 权限** - 使用只读/受限的 API Key
2. **定期检查记忆文件** - `~/.openclaw/workspace/memory/`
3. **不使用 IMA 时** - 不配置 `~/.config/ima/` 凭据
4. **沙箱测试** - 首次使用时先用测试对话验证行为

### 配置文件位置

| 文件 | 用途 | 必需 |
|------|------|------|
| `llm_config.json` | LLM/Embedding API 配置 | 必需 |
| `~/.config/ima/` | IMA 云同步凭据（可选）| 可选 |
| `~/.openclaw/credentials/secrets.env` | OpenClaw 全局凭据 | 必需 |
| `~/.openclaw/workspace/memory/` | 本地记忆存储 | 必需 |

### 访问的全局路径

本 skill 会读写以下全局路径：
- `~/.openclaw/` - OpenClaw 工作区
- `~/.openclaw/workspace/memory/` - 记忆文件
- `~/.openclaw/credentials/secrets.env` - 凭据
- `~/.config/ima/` - IMA 云同步（可选）

## 🔑 API Key 配置教程

### 获取 LLM API Key（模力方舟）

1. 访问 [模力方舟](https://fangzhou.top)
2. 注册/登录后进入控制台
3. 左侧菜单 → API Key → 创建新密钥
4. 复制 Key 并填入配置

**推荐配置：**
- Provider: `minimax`
- Model: `MiniMax-M2.7-highspeed`
- Base URL: `https://api.minimaxi.chat/v1`

### 获取 Embedding API Key

1. 访问 [Gitee AI](https://ai.giffox.com)
2. 注册后创建多年免费 API Key
3. 填入配置

**配置位置：** `~/.openclaw/credentials/secrets.env`

```
LLM_API_KEY=你的模力方舟API Key
EMBEDDING_API_KEY=你的Gitee AI Key
```

> ⚠️ API Key 安全性很重要，切勿泄露或提交到 GitHub

### 🎯 核心原则

1. **能不说就不说** - 执行比解释更重要

4. **引导不强迫** - 提供选项，不代替选择


### 🔧 自修复系统

当检测到问题时，AI 可自动修复以下常见问题：

| 问题 | 检测 | 自动修复 |
|------|------|----------|
| Embedding缓存损坏 | JSON解析失败 | 删除缓存文件 |
| 配置文件损坏 | JSON解析失败 | 备份+重建 |
| Feature Flag损坏 | JSON解析失败 | 重置 Flags |
| 记忆目录缺失 | 目录不存在 | 创建目录 |

**使用方式：**
| 用户说 | AI 行动 |
|--------|----------|
| "自修复" / "修复系统" | 执行 `auto_fixer.py fix` |
| "检查问题" / "诊断" | 执行 `auto_fixer.py diagnose` |

**AI 自动修复流程：**
```
检测到问题 → 诊断原因 → 评估能否自修复
  ↓
能修复 → 自动修复 → 报告结果
  ↓
不能修复 → 提示用户 → 引导解决方案
```

**自修复示例：**
```
用户: 系统好像有问题
AI: 🔧 检测到问题...
   ⚠️ Embedding 缓存文件损坏
   执行自动修复...
   ✅ 已修复：删除损坏的缓存文件
   ✅ 系统已恢复正常
```


### ⏰ 定时自检自修复

系统会不定时自动进行健康检查和修复：

#### 触发时机

| 触发条件 | 检查深度 | 耗时 |
|----------|----------|------|
| 每次AI响应前 | 快速检查 | <1秒 |
| 空闲超过1小时 | 完整检查 | 3-5秒 |
| 记忆操作失败时 | 针对性检查 | <2秒 |
| 定时任务触发 | 完整检查 | 3-5秒 |

#### 检查项目

| 检查项 | 正常表现 | 异常处理 |
|--------|----------|----------|
| Embedding缓存 | JSON有效 | 删除重建 |
| 配置文件 | JSON有效 | 备份+重建 |
| Feature Flag | JSON有效 | 重置默认值 |
| 记忆目录 | 存在 | 自动创建 |
| 磁盘空间 | >100MB | 提示清理 |

#### 修复流程

```
┌─────────────────────────────────────┐
│           检测到问题                  │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│     评估能否自动修复                 │
└─────────────────┬───────────────────┘
                  ▼
        ┌───────┴───────┐
        ▼               ▼
   能修复            不能修复
     │               │
     ▼               ▼
  自动修复         记录日志
  不打扰          提示用户
     │               │
     └───────┬───────┘
             ▼
      报告结果
   （仅用户询问时）
```

#### 静默原则
