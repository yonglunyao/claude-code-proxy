# 记忆生命周期管理

本文档详细说明记忆在各层级之间的流转规则。

## 记忆生命周期状态

```
[创建] → [短期] → [中期] → [长期] → [档案]
           ↓         ↓         ↓         ↓
         [遗忘]   [清理]    [归档]    [永久]
```

## 状态转换触发条件

### 短期 → 中期

**触发条件**：
- 会话结束前，有价值的信息
- 用户明确要求记录
- 完成重要任务

**执行方式**：
- 写入 `memory/YYYY-MM-DD.md`
- 添加适当标签

### 中期 → 长期

**触发条件**：
- 信息被检索 ≥3 次
- 用户说"记住这个"、"以后都这样"
- 决策被反复引用
- 错误教训需要长期避免
- 定期审查时识别为重要

**执行方式**：
- 运行 `scripts/promote.py`
- 或手动复制到 MEMORY.md
- 在原文件标记 `[已升级]`

### 长期 → 档案

**触发条件**：
- 重要决策需要永久保存
- 技术知识需要跨设备访问
- 用户要求备份

**执行方式**：
- 运行 `scripts/sync_ima.py`
- 或手动上传到 IMA 知识库

## 清理规则

### 中期记忆清理

| 条件 | 操作 |
|------|------|
| 超过30天且无重要标记 | 删除 |
| 超过30天但有重要标记 | 归档到 archive/ |
| 已升级到长期 | 可删除 |

### 长期记忆审查

每月执行一次：
1. 检查 MEMORY.md 中的过时信息
2. 标记为 `[已过期]` 或删除
3. 更新冲突信息
4. 清理已完成的待处理事项

## 冲突解决

当新旧信息冲突时：
1. **时间优先**：新信息覆盖旧信息
2. **标记旧信息**：添加 `[已过期 - YYYY-MM-DD]`
3. **记录变更**：在更新日志中记录

示例：
```markdown
- 技术栈：React（已过期 - 2024-01-15）
- 技术栈：Vue 3（当前）
```

## 记忆质量标准

### 应该记录的内容

- ✅ 用户明确表达的偏好
- ✅ 重要决策及其理由
- ✅ 踩过的坑和解决方案
- ✅ 反复使用的知识
- ✅ 待办事项和承诺

### 不应该记录的内容

- ❌ 闲聊内容
- ❌ 临时性的、一次性的信息
- ❌ 用户明确要求不记录的内容
- ❌ 敏感信息（密码、密钥等）

## 记忆索引

MEMORY.md 应包含索引，方便快速定位：

```markdown
## 📌 记忆索引

| 标签 | 说明 |
|------|------|
| #用户 | 用户信息、偏好、习惯 |
| #技术 | 技术决策、实现、经验 |
| #决策 | 重要决策及理由 |
| #错误 | 错误教训、踩坑记录 |
| #待处理 | 等待执行的任务 |
```
