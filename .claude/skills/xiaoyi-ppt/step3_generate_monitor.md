# 子流程三：调用云服务 & 监控 & 交付

> **模型须知**：进入此子流程前必须先完整阅读本文件，再执行任何操作。
> 前置条件：`$OUTLINE_URL` 已在子流程二中成功获取。

---

## 步骤一：调用 PPT 生成脚本

**目标**：将用户原始指令和大纲文件 URL 作为参数，在后台启动 PPT 生成任务。

> **重要**：将 exec 工具的 `yieldMs` 设置为 `2400000`（40 分钟）再执行此步骤。

```bash
PPT_SESSION_ID="$PPT_SESSION_ID" \
  $PYTHON_CMD "{baseDir}/scripts/generate_ppt.py" \
  "<用户的完整原始指令>" \
  --files "[\"$OUTLINE_URL\"]" \
  > /dev/null 2>&1 &

PPT_PID=$!
echo "PPT 生成任务已启动"
echo "PID：$PPT_PID"
echo "日志：$PPT_SESSION_DIR/generate.log"
```

**关于 `<用户的完整原始指令>`**：
- 忠实还原用户的所有需求，包括：主题、页数限制、目标受众、风格偏好、重点内容等
- 不要截断或缩减，完整传递给云端服务

**完成汇报**：
```
✅ PPT 生成任务已启动，开始监控进度...
```

---

## 步骤二：监控进度

**目标**：每 15 秒检查一次进程状态和日志，向用户持续汇报进展。

### 2.1 监控循环（每 15 秒执行一次，最多 80 次）

初始化计数器后开始轮询：

```bash
PPT_POLL_COUNT=0
PPT_POLL_MAX=80
```

每轮执行：

```bash
PPT_POLL_COUNT=$((PPT_POLL_COUNT + 1))

# 检查进程是否仍在运行
kill -0 $PPT_PID 2>/dev/null && PPT_ALIVE=true || PPT_ALIVE=false

# 读取最新日志（最后 5 行，了解最新进度，避免读取太多导致上下文干扰）
tail -5 "$PPT_SESSION_DIR/generate.log"
```

等待 15 秒后重复，直到满足终止条件。**当 `PPT_POLL_COUNT` 达到 `PPT_POLL_MAX`（80 次）时，即使进程仍在运行，也停止轮询，向用户报告超时，并告知日志路径供手动查看。**

### 2.2 日志解读与汇报

根据日志内容，向用户汇报当前阶段：

| 日志包含 | 向用户汇报 |
|---------|-----------|
| `[进度]` | 显示进度信息内容 |
| `[调用工具]` | 「正在调用工具」 |
| `[进行中]` | 「正在执行中」 |
| `[生成文件]` | 「文件已生成：{filename}」 |
| `[文件已保存]` | 「文件已下载到本地：{path}」 |
| `[DONE]` | 「PPT 生成完成，停止监控，进入步骤三...」 |

### 2.3 终止条件判断

| 状态 | 处理方式 |
|------|---------|
| 进程运行中，且未超时 | 继续等待 15 秒，再次检查 |
| 进程已退出 + 日志含 `[DONE]` | **停止轮询，进入步骤三** |
| 进程已退出 + 日志含 `[ERROR]` | 向用户报告错误信息，停止 |
| 进程已退出 + 无任何终止标记 | 向用户报告异常崩溃，询问是否重试 |
| 轮询次数达到 80 次（约 20 分钟） | 停止轮询，向用户报告超时，并告知日志路径：`$PPT_SESSION_DIR/generate.log` |

## 步骤三：交付结果

**目标**：将生成的 PPT 文件传送给用户。

### 3.1 查找生成文件

```bash
ls -la "$PPT_SESSION_DIR"
```

从日志中提取：
- **下载 URL**：`[生成文件]` 行中的 URL
- **本地路径**：`[文件已保存]` 行中的绝对路径

### 3.2 发送 PPT 给用户

向用户提供完整的交付信息：

```
✅ PPT 生成完成！

📎 本地路径：/tmp/xiaoyi_ppt/<session_id>/<filename>.pptx
🔗 下载链接：<url>

【内容简述】<简要说明 PPT 的主题、章节结构和主要内容>
```
