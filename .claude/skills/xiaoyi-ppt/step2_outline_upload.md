
---

## 步骤二：编写大纲内容

**核心原则**：
- 内容 100% 源于子流程一收集的信息，**严禁编造**
- 标题与大纲一一对应，严禁合并、遗漏、新增
- 用户的具体要求（页数、重点、风格）优先

**内容填充规则**：
- 有图片或表格的页面：文字 ≤200 字
- 无图表的页面：文字 ≤400 字
- 每页 ≤1 张图片
- 表格：≤8 行 × 6 列
- 数学公式：使用 LaTeX 格式
- 每页划分为 2-4 个独立段落/模块，小标题**加粗**，合理使用列表

**图片搜索查询**：每个内容页末尾增加 `<image_search_queries>` 部分，包含 ≤3 个搜索关键词：
- 关键词应具体、有针对性
- 优先使用中文关键词
- 已有原文图片的页面可不写

---

## 步骤三：格式规范

每页之间用 `\n---\n` 分隔，页面层级标记：
- `# 类型` 开头 — 类型声明页（第一页，含 `<category>` 标签）
- `# 标题` — 封面页（第二页）
- `## ...目录` — 目录页
- `## 01 章节标题` — 章节页（**仅双层结构**）
- `### 内容标题` — 内容页

**绝对不可以遗漏 `\n---\n` 分割线，否则会导致格式解析失败，任务无法完成**

## 步骤五：上传大纲文件

**目标**：将 `outline.md` 上传到云端，获取可供 PPT 生成脚本访问的 URL。

> ⚠️ **必须使用 `{baseDir}/scripts/upload_file.py` 执行上传，严禁调用任何其他 skill 或工具代替。** 其他上传方式生成的 URL 格式与 PPT 生成服务不兼容，会导致任务失败。

```bash
OUTLINE_URL=$($PYTHON_CMD "{baseDir}/scripts/upload_file.py" "$PPT_SESSION_DIR/outline.md")

if [ $? -ne 0 ] || [ -z "$OUTLINE_URL" ]; then
  echo "错误：大纲文件上传失败，请检查网络连接和 .xiaoyienv 配置"
  exit 1
fi

echo "大纲文件 URL：$OUTLINE_URL"
```

**完成汇报**：
```
✅ 大纲文件上传成功，开始生成 PPT...
```
