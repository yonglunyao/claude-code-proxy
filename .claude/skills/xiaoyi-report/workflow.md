## 步骤 1：需求澄清

**目标：** 明确研究方向和约束条件

**操作：**
1. 询问研究目标：学习了解 / 做决策 / 撰写内容 / 商业分析
2. 确认约束：时间范围、地域焦点、深度级别、输出格式

**快速响应：** 用户说"直接研究"则使用默认设置继续

**完成后更新：** `- [x] 1. 需求澄清`

---

## 根据研究类型调整流程

在步骤1明确研究目标后，根据用户需求调整执行深度：

| 类型 | 研究维度 | 来源数 | 报告长度 | 触发词示例 |
|------|----------|--------|----------|------------|
| 快速概览 | 3-5个 | 10-15个 | ~2000字 | "快速了解"、"简单调研"、"大概看一下" |

## 步骤 2：研究规划

**目标：** 构建研究框架，规划搜索策略

**操作：**
1. 拆分主题为5-8个核心研究维度
2. 为每个维度设计2-3个搜索关键词变体
3. 确定信息源优先级

**研究维度模板：**
```
├─ 📊 现状分析：当前状态如何？核心数据？
├─ 🔍 技术原理：底层机制/技术是什么？
├─ 🏢 主要玩家：谁是关键参与者？
├─ 📈 趋势分析：发展方向？未来预测？
├─ ⚖️ 挑战与风险：面临哪些问题？
├─ 💡 机遇与前景：有哪些潜在机会？
└─ 🌍 区域差异：不同地区/市场的差异？
```

**完成后更新：** `- [x] 2. 研究规划`

## 步骤 3：信息搜索与提取

**目标：** 逐个维度执行搜索，主动提取关键信息

### 3.1 执行流程

**⚠️ 关键：不只是执行搜索命令，必须主动完成信息提取全流程：**

对每个研究维度，依次执行以下步骤：

1. **执行搜索** → 调用搜索脚本获取结果列表
2. **分析结果** → 从搜索结果中筛选2-3个高价值来源（优先学术/官方来源）
3. **深度获取** → 使用 WebFetch 访问并阅读筛选出的来源内容
4. **提取信息** → 从来源中提取：核心数据、关键观点、利益相关者、争议点
5. **结构化输出** → 整理为JSON格式，继续下一个维度

### 3.2 搜索脚本

```bash
node /home/sandbox/.openclaw/workspace/skills/xiaoyi-web-search/scripts/search.js "[关键词]" -n 8
```

### 3.3 信息提取要求

**禁止只执行搜索命令！** 必须从搜索结果中：
- 筛选高价值来源（优先学术论文、官方报告、权威媒体）
- 使用 WebFetch 深度获取来源内容
- 提取核心数据、关键观点、主要参与者、争议点

### 3.4 输出格式

每个维度完成后，整理为以下格式：

```json
{
  "维度": "[维度名称]",
  "关键发现": [{"发现": "...", "数据": "...", "来源": "url", "置信度": "高/中/低"}],
  "核心数据": [...],
  "来源列表": [{"标题": "...", "url": "...", "可信度": "⭐⭐⭐"}]
}
```

### 3.5 执行顺序示例

```
# 按顺序逐个维度执行
1. 现状分析：搜索 → 筛选来源 → 提取信息 → 整理输出
2. 技术原理：搜索 → 筛选来源 → 提取信息 → 整理输出
3. 市场格局：搜索 → 筛选来源 → 提取信息 → 整理输出
4. 挑战风险：搜索 → 筛选来源 → 提取信息 → 整理输出
5. 趋势预测：搜索 → 筛选来源 → 提取信息 → 整理输出
```

## 步骤 4：交叉验证

**目标：** 验证关键数据，识别矛盾

**操作：**
1. 对关键数据点进行跨维度验证
2. 构建验证矩阵
3. 标注数据置信度

**验证矩阵格式：**
| 数据项 | 来源A | 来源B | 验证结果 |
|--------|-------|-------|----------|
| 市场规模 | $X亿 | $Y亿 | ✅一致 / ⚠️差异 |

**完成后更新：** `- [x] 4. 交叉验证`

- 风险评估：技术/市场/政策风险
