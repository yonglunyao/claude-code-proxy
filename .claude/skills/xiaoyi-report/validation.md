# 验证规则与质量准则

> 本文件定义数据验证规则和质量标准。按需阅读。

---

## 数据验证规则

### 多源验证要求

| 数据类型 | 最低验证要求 |
|----------|--------------|
| 核心数据（市场规模、增长率等） | 至少2个独立来源 |
| 技术参数 | 1个权威来源 |
| 观点/预测 | 标注来源和置信度 |

### 交叉验证流程

1. 收集同一数据点的多个来源
2. 比对数据一致性
3. 标注验证结果：✅一致 / ⚠️差异 / ❌矛盾
4. 差异较大时，使用范围值并说明差异原因

## 来源可信度分级

| 等级 | 来源类型 | 标记 |
|------|----------|------|
| 最高 | 学术论文、官方报告、行业白皮书、统计局数据 | ⭐⭐⭐ |
| 高 | 权威媒体（财新、彭博、路透社）、行业分析报告 | ⭐⭐ |
| 中 | 专业博客、技术文档、专家评论 | ⭐ |
| 低 | 论坛讨论、社交媒体、自媒体文章 | ⚠️（需验证）|

---

## 置信度评级标准

| 等级 | 标准 | 标记 |
|------|------|------|
| 高 | 多个独立权威来源一致确认 | 🟢 |
| 中 | 有可靠来源，但来源较少或存在轻微矛盾 | 🟡 |
| 低 | 仅单一来源或来源可信度有限 | 🔴 |

## 引用规范

### 正确示例
```markdown
市场份额达到 35%（来源：IDC 2024年度报告）
专家认为该技术仍处于早期阶段 [1]
```

### 错误示例
```markdown
市场份额很大（无来源）❌
该技术已经成熟（来源：一篇博客文章，未经验证）❌
```

## 质量审查清单

发布前检查：

- [ ] **数据准确性**：所有数据点都有来源引用
- [ ] **时效性**：优先使用最近12个月的数据
- [ ] **平衡性**：呈现不同观点，不偏颇
- [ ] **完整性**：覆盖所有研究维度
- [ ] **可读性**：结构清晰，层次分明
- [ ] **可追溯性**：所有关键论断可追溯至原始来源
- [ ] **不确定性标注**：对不确定信息明确标注置信度

---

## 零容忍规则

- ❌ 禁止编造数据
- ❌ 禁止引用不可靠来源作为核心证据
- ❌ 禁止忽略明显的矛盾数据
- ❌ 不确定时必须明确说明"未找到足够数据"
